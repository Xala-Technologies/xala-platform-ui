## Accessibility

### Keyboard Interaction

| Key | Action |
|-----|--------|
| `Tab` | Move focus to button |
| `Enter` | Activate button |
| `Space` | Activate button |

### Screen Reader

Buttons are announced as "button" role. Include descriptive text.

```tsx
// Good - clear action
<Button>Save Changes</Button>

// Avoid - unclear
<Button>Click Here</Button>
```

### Focus

Focus ring is visible on keyboard focus. Do not remove or hide it.

## Do and Don't

### Do

- Use action verbs (Save, Submit, Delete)
- Keep text concise (2-3 words)
- Use Primary for the main action
- Add icons for visual clarity

### Don't

- Use multiple Primary buttons in one section
- Disable without explanation
- Use vague text like "Click Here"
- Mix button styles inconsistently

## data-testid Convention

```tsx
<Button data-testid="save-button">Save</Button>
<Button data-testid="delete-button">Delete</Button>
<Button data-testid="cancel-button">Cancel</Button>
```

## Migration Notes

If migrating from custom buttons:

1. Replace custom button components with DS `Button`
2. Map old variants to `primary`, `secondary`, `tertiary`
3. Replace custom icons with DS icon registry
4. Remove any inline color styles
